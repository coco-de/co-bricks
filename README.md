## Usage

### Synchronize App Bricks

```sh
# Synchronize app bricks (automatically detects .envrc from current directory)
$ co_bricks sync --type app

# Synchronize app bricks from specific project directory
$ co_bricks sync --type app --project-dir /path/to/project
```

### Synchronize Monorepo Brick

```sh
# Synchronize monorepo brick (automatically detects .envrc from current directory)
$ co_bricks sync --type monorepo

# Synchronize monorepo brick from specific project directory
$ co_bricks sync --type monorepo --project-dir /path/to/project
```

### Other Commands

```sh
# Show CLI version
$ co_bricks --version

# Show usage help
$ co_bricks --help

# Show sync command help
$ co_bricks sync --help
```

## How It Works

1. **Automatic .envrc Detection**: The CLI automatically searches for `.envrc` file starting from the current directory (or specified `--project-dir`) and walks up the directory tree until found.

2. **Project Configuration**: The `.envrc` file is parsed to extract:
   - `PROJECT_NAME`: Project name (e.g., `good_teacher`)
   - `ORG_NAME`: Organization name (e.g., `laputa`)
   - `ORG_TLD`: Organization TLD (e.g., `im`)
   - And other configuration values

3. **Template Synchronization**:
   - **App Type**: Synchronizes `template/{project}/app/` â†’ `bricks/{app,console,widgetbook}/__brick__/`
   - **Monorepo Type**: Synchronizes `template/{project}/*` â†’ `bricks/monorepo/__brick__/{{project_name}}/`

4. **Template Variable Conversion**: All hardcoded project names, organization names, Firebase IDs, etc. are automatically converted to Mason template variables (e.g., `{{project_name.snakeCase()}}`).

## Requirements

- Dart SDK ^3.9.0
- A project with `.envrc` file containing project configuration
- Template projects in `template/{project_name}/` directory
